# CareView AI Healthcare Analytics Dashboard

## Simplified Documentation

---

## 1. Executive Summary

The CareView AI Healthcare Analytics Dashboard is a healthcare analytics platform designed to convert hospital data into meaningful insights. The system helps healthcare organizations monitor patient activity, department performance, financial metrics, and disease trends.

The dashboard is built using Snowflake for data storage and processing, Snowflake Cortex AI for generating clinical insights, and Streamlit for creating an interactive visualization dashboard. Plotly is used for creating charts and visualizations.

This platform supports decision-making for multiple hospital stakeholders including executives, clinical leaders, department managers, finance teams, and care coordinators. The dashboard provides a clear overview of hospital performance and helps identify operational, clinical, and financial improvements.

---

## 2. Project Overview

The CareView system follows a Medallion Architecture, which organizes data processing into multiple layers. Each layer improves the quality and usability of the data.

Data first enters the system in its original form and then moves through cleaning, transformation, and aggregation stages before being used for analytics and AI insights.

The architecture consists of the following layers:

### Bronze Layer (Raw Data)

The Bronze layer stores the original raw healthcare data without modification. This includes tables such as patients, visits, diagnoses, and billing records. The purpose of this layer is to preserve the original data exactly as received from the source systems.

### Silver Layer (Cleaned Data)

The Silver layer contains cleaned and validated data. In this stage, errors are corrected, duplicates are removed, and data formats are standardized. This layer ensures that the data is reliable for analysis.

### Gold Layer (Analytics Layer)

The Gold layer contains aggregated and business-ready data. This layer organizes data into tables designed specifically for reporting and analytics. Examples include patient summaries, department metrics, diagnosis analytics, and monthly revenue data.

### AI-Ready Layer

The AI-Ready layer enhances the Gold layer data using Snowflake Cortex AI. This layer generates clinical insights such as treatment recommendations, diagnosis severity evaluation, and priority classification.

Finally, the data from these layers is displayed through the Streamlit dashboard, which allows users to interact with the analytics visually.

---

## 3. Data Architecture

The AI-Ready layer contains AI-generated insights related to clinical diagnoses. Each record represents a diagnosis and includes information such as the diagnosis code, diagnosis name, number of cases, number of affected patients, severity level, number of critical cases, and AI-generated clinical insights.

The table also includes a column called clinical priority, which categorizes diagnoses into high priority, standard priority, or low priority depending on the severity and risk level.

The Gold layer contains several aggregated tables used by the dashboard.

### Patient Summary Table

The Patient Summary table includes demographic information such as patient ID, age, gender, and blood type. It also contains visit history, clinical information such as number of diagnoses and severity levels, and financial details such as billed amount, paid amount, and outstanding balance.

### Department Metrics Table

The Department Metrics table summarizes operational performance for each department. It includes total visits, number of unique patients, average length of stay, and total revenue generated by the department.

### Monthly Revenue Table

The Monthly Revenue table tracks financial performance over time. It includes billing amounts, insurance payments, patient payments, pending payments, and denied claims for each department and month.

### Diagnosis Analytics Table

The Diagnosis Analytics table provides insights into disease patterns. It contains diagnosis codes, diagnosis names, severity levels, and case counts.

## 4. Dashboard Pages

The CareView dashboard contains six major analytical sections.

### Executive Overview

The Executive Overview page provides a quick summary of overall hospital performance. This page allows leadership teams to quickly understand the current status of operations.

The page displays key metrics including the total number of patients, total revenue generated, total patient visits, and the average severity of diagnoses across the hospital.

Visualizations on this page include a revenue distribution chart showing how revenue is divided across departments, a visit volume chart displaying the number of patient visits per department, and a monthly trend chart that illustrates financial performance over time.

These visualizations help hospital executives quickly understand which departments are performing well and how financial performance is evolving.

### AI Clinical Intelligence

The AI Clinical Intelligence page displays insights generated using Snowflake Cortex AI. The purpose of this page is to assist clinical staff in identifying important diagnoses and treatment considerations.

Diagnoses are categorized based on priority levels. High-priority diagnoses are highlighted so that medical teams can identify conditions requiring immediate attention.

The dashboard also includes a distribution chart showing the proportion of diagnoses categorized as high priority, standard priority, or low priority.

Another visualization compares diagnosis severity with case volume. This chart helps identify conditions that are both severe and common.

Users can also select a specific diagnosis to view AI-generated clinical insights, which may include treatment suggestions, preventive recommendations, or risk factors.

Priority classification is determined using severity levels and the number of critical cases associated with each diagnosis.

### Department Performance

The Department Performance page analyzes operational metrics for each hospital department.

This page helps department managers understand how efficiently their departments are operating.

Metrics include total patient visits, number of unique patients, average length of stay, and total revenue generated by the department.

The page also compares departments based on visit volume and financial performance. Another visualization shows the distribution of visit types, which include emergency visits, inpatient admissions, and outpatient appointments.

The average length of stay metric helps identify departments where patient discharge processes could be improved.

### Financial Analytics

The Financial Analytics page focuses on revenue cycle management.

This page displays financial metrics including total billed amount, collected payments, insurance reimbursements, pending payments, and collection rate.

The collection rate represents the percentage of billed revenue that has been successfully collected. It is calculated by dividing the total payments received by the total billed amount.

The dashboard also displays monthly revenue trends to help finance teams identify seasonal patterns or payment delays.

Another chart compares collection rates across departments. This helps identify departments that may need improvements in billing processes or insurance claim management.

A payment source distribution chart shows the proportion of revenue coming from direct patient payments, insurance reimbursements, pending payments, and denied claims.

### Disease Analytics

The Disease Analytics page analyzes diagnosis trends across the patient population.

This page identifies the most common diagnoses and displays them using a ranking chart. It also shows the distribution of severity levels across all diagnoses.

A severity heatmap highlights which diagnoses contribute most to severe or critical cases. This analysis helps hospitals allocate resources effectively and plan preventive care programs.

Diagnosis codes follow the ICD-10 classification system, which is a global medical coding standard. Example diagnoses include respiratory infections, hypertension, heart attacks, diabetes, lung cancer, and musculoskeletal disorders.

Severity levels range from mild conditions that require minimal treatment to critical conditions that require immediate medical intervention.

### Patient Insights

The Patient Insights page focuses on demographic analysis and patient behavior patterns.

This page displays the distribution of patient ages, allowing healthcare providers to understand the dominant age groups receiving care.

The gender distribution chart shows the balance between male and female patients in the system.

Blood type distribution is also analyzed because it is important for transfusion planning and emergency preparedness.

The dashboard also identifies patients with high severity scores, which helps clinicians track high-risk individuals.

A patient search feature allows users to quickly find patient information using patient ID, first name, or last name.

## 7. Performance Optimization

Performance is optimized through data caching and efficient query design.

The dashboard uses Streamlit's caching feature to temporarily store query results. This reduces repeated database calls and improves dashboard loading speed. Cached data is refreshed every five minutes.

Additionally, the Gold layer contains pre-aggregated tables. This reduces computation time when generating charts and metrics.

---

## 8. Security

Security is implemented through Snowflake's Role-Based Access Control (RBAC). This ensures that only authorized users can access specific data.

Database connections use secure sessions, and sensitive data is not exposed directly in the dashboard interface. Snowflake governance tools provide logging and monitoring for auditing purposes.
